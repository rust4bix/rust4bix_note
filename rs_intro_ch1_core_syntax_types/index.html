Core Syntax & Types

Why Syntax Matters in Bioinformatics

In bioinformatics, we routinely process gigabyte–terabyte scale data (FASTQ, BAM, VCF). Small inefficiencies in syntax or type usage can translate into hours of runtime or gigabytes of wasted memory.

Rust’s syntax is not just aesthetic—it enforces:

  • Memory safety (no silent bugs)

  • Explicit data handling

  • High performance without a garbage collector

This chapter builds the foundation for writing correct and fast genomic pipelines.

Variables and Immutability

Rust variables are immutable by default.

let x = 10;
// x = 20; // error

If you run the Rust code below: 

let x = 10;
x = 20; // error

You will get the error message shown below: 

[E0384] Error: cannot assign twice to immutable variable `x`
   ╭─[ command_2:1:1 ]
   │
 1 │ let x = 10;
   │     ┬  
   │     ╰── first assignment to `x`
   │     │  
   │     ╰── help: consider making this binding mutable: `mut `
 2 │ x = 20; // error
   │ ───┬──  
   │    ╰──── cannot assign twice to immutable variable
   │ 
   │ Note: You can change an existing variable to mutable like: `let mut x = x;`
───╯
[unused_assignments] Error: value assigned to `x` is never read
   ╭─[ command_2:1:1 ]
   │
 1 │ let x = 10;
   │         ─┬  
   │          ╰── warning: value assigned to `x` is never read
───╯

Mutable variables

let mut count = 0;
count += 1;
println!("count: {}", count);

count: 1

Use mut only when necessary—this reduces bugs and improves reasoning.

Shadowing (Rebinding Variables)

let x = 5;
let x = x + 1;
let x = x * 2;
println!("x: {}", x);

x: 12

Unlike mutation, shadowing creates a new variable.

let seq = "acgt";
let seq = seq.to_uppercase();
println!("seq: {}", seq);

seq: ACGT

You transform data without mutating the original reference.

Scalar Types

Integers

let x: i32 = 10;
let y: u64 = 100;

Use cases:

  • u64 → genome positions

  • i32 → score differences

Floating point

let gc_ratio: f64 = 0.42;

Boolean

let is_mapped = true;
let is_cell = false;
println!("is_mapped: {}", is_mapped);
println!("is_cell: {}", is_cell);

is_mapped: true
is_cell: false
